LG Regen

동남아 이슬람 국가 맞춤 ThinQ 활용 솔루션
동영상
산출문서 GitHub (Front-end) GitHub (Back-end)

동남아(인도네시아·말레이시아) 무슬림 가정의 생활 패턴을 반영해 가전 제어 경험을 재구성한 스마트홈 솔루션입니다.
Flutter 앱–Spring Boot–MySQL 기반으로 전력(선불·후불) 대시보드 및 사용량 저장 기능을 구현했고, 홈 화면에서 날씨 + 전력 정보를 통합 제공하도록 API를 설계했습니다.

핵심 성과
  • REST API 6개 구현 (로그인/홈 통합/선불/후불/사용량)
  • 전력 도메인 엔티티 7개 설계 (Users, Meters, Bills, Usage_Logs, Product, Prepaid_Meters, Token_Top_Up)
  • 일일 사용량은 날짜 기준 Upsert 처리로 중복 저장 방지 및 최신 값 유지
  • 선불/후불 계량기 미등록 상황을 고려해 대시보드 응답 예외 처리 적용
프로젝트 요약
한줄 소개 동남아 무슬림 가정 패턴을 반영한 스마트홈/전력관리(선불·후불) 솔루션
구조 Flutter(Client) ↔ Spring Boot(API) ↔ MySQL(DB) / Naver Cloud(배포 고려) / ESP32+LCD(연동 고려)
담당 역할
  • 백엔드 REST API 설계/구현 (선불·후불 대시보드, 사용량 저장, 로그인)
  • 전력 도메인 데이터 모델링(ERD) 및 CRUD 흐름 구성
  • 홈 대시보드에서 날씨/전력 데이터 통합 응답 설계
기술 스택 Spring Boot, MySQL, Flutter, Git/GitHub, (NCP 배포 고려), ESP32+LCD(연동 고려)
증빙 시스템 구조도, ERD, 기능 맵, 앱 화면 캡쳐, Controller/API 코드
역할 · 업무 · 성과
백엔드 API 선불(충전/대시보드)·후불 대시보드·일일 사용량 저장·로그인·홈 통합 API 구현
✔ API 6개 구현
데이터 정합성 Flutter가 전송한 오늘 사용량을 날짜 기준으로 저장/갱신(Upsert)하여 중복 저장을 방지
통합 응답 설계 홈 화면에서 사용자 정보 + 위치 기반 날씨 + 전력(선불/후불) 대시보드를 단일 응답(DashboardResponseDTO)로 통합
안정성 선불/후불 계량기 미등록 시에도 대시보드 응답이 깨지지 않도록 예외 처리 적용
DB/도메인 설계 사용자–계량기–선불충전–사용량–청구서–제품 엔티티 관계 설계
✔ 엔티티 7개 설계
REST API 목록 (코드 기반 확정)

Prepaid POST /api/users/{userId}/prepaid-topup — 선불 토큰 충전(설정 완료/저장)
Prepaid GET /api/users/{userId}/prepaid-dashboard — 선불 대시보드 조회(잔액/예상 사용기간)
Postpaid GET /api/users/{userId}/postpaid-dashboard — 후불 대시보드 조회(예상요금/사용량/청구내역)
Usage POST /api/users/{userId}/usage/today — 오늘 사용량 저장(날짜 기준 Upsert)

PROJECT II

Docker Container Monitoring System

November 2024 – February 2025

cAdvisor · Prometheus · Grafana 기반 컨테이너 모니터링
발표자료 논문(PDF)

Docker 환경에서 실행되는 컨테이너의 CPU · Memory · Network 사용량을 실시간으로 수집(cAdvisor) → 저장(Prometheus) → 시각화(Grafana) 하는 모니터링 시스템입니다.
Docker Compose로 실행 환경을 표준화하여, 운영 관점에서 컨테이너별 리소스 변화를 빠르게 파악할 수 있도록 구성했습니다.

핵심 성과
  • Docker Compose로 모니터링 스택을 일괄 배포 (cAdvisor/Prometheus/Grafana)
  • 컨테이너별 CPU/Memory/Network 지표를 대시보드로 실시간 확인
  • 부하 컨테이너(task0~task2)로 지표 변화 검증 (CPU/메모리/트래픽 유발)
프로젝트 요약
한줄 소개 Docker 컨테이너 리소스(CPU/Memory/Network)를 수집·저장·시각화하는 모니터링 시스템
구조 cAdvisor(수집) → Prometheus(저장) → Grafana(시각화) / Docker Compose로 구성
담당 역할
  • 모니터링 스택 설계(역할 정의/데이터 흐름 설계)
  • Docker Compose 기반 배포/실행 구성 표준화
  • Prometheus 스크래핑 타깃/주기 설정 구성
  • Grafana 대시보드 구성(CPU/Memory/Network 중심)
기술 스택 Ubuntu 24.04 LTS, Docker Engine/Compose, cAdvisor, Prometheus, Grafana
설계 · 기능 · 화면 시스템 구조도, Grafana 대시보드 캡쳐, VM vs Docker 비교 자료
역할 · 업무 · 성과
수집 cAdvisor로 컨테이너 단위 CPU/Memory/Network 메트릭 수집
저장 Prometheus가 cAdvisor 엔드포인트를 주기적으로 스크래핑하여 시계열 데이터 누적
시각화 Grafana에서 컨테이너별 지표를 대시보드로 구성하여 운영 관점(병목/이상징후)을 빠르게 확인
✔ Grafana 대시보드 동작 확인
검증 부하 컨테이너(task0~task2)로 CPU/메모리/트래픽 변화가 대시보드에 반영되는지 검증
증빙 캡쳐 (구조 · 동작)
개발환경

  • Docker Engine / Docker Compose
